## 2024-05-15 - Interactive vs Static Read-Only Component A11y (StarRating)
**Learning:** Components that handle both read-only and interactive visual states often have poor screen-reader flow if both utilize the same HTML structure. Screen-readers announce interactive `buttons` nicely, but if those buttons are disabled for read-only modes, the screen-reader unnecessarily announces multiple disabled controls or icons sequentially (e.g., "star graphic, star graphic, star graphic").
**Action:** Always fork the ARIA structure for multi-mode components. Use a single container with `role="img"` and a summary `aria-label` (e.g. "Rating: 4 out of 5 stars") while hiding individual children (`aria-hidden="true"`) for read-only modes. For interactive modes, use proper `role="radiogroup"` or `button` lists with full focus and `aria-pressed`/`aria-checked` bindings.
