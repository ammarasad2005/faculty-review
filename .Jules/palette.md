## 2024-04-27 - Star Rating Accessibility Pattern
**Learning:** For components that have both a read-only and an interactive mode (like star ratings), the screen reader experience must be handled differently for each mode to avoid spammy readouts. In read-only mode, individual stars should be hidden (`aria-hidden="true"`) and the container should use `role="img"` with an `aria-label` summarizing the state (e.g., "4 out of 5 stars"). In interactive mode, individual stars must have their own `aria-label` and `aria-pressed` states so the user understands they are actionable.
**Action:** When auditing or building multi-mode UI components, always split the accessibility strategy. Never apply interactive ARIA patterns (like `aria-pressed`) to read-only views, and conversely, ensure read-only views provide a concise summary rather than forcing screen readers to parse visual implementation details.
